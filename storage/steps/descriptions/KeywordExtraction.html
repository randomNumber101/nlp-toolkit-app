üõ† How the Keyword Extraction Operation Works

  • Text Processing: The operation extracts the most relevant keywords from a selected text column.
  • spaCy NLP Model: Uses a pre-trained spaCy language model to tokenize text and identify important words.
  • Filtering:
    • Only alphabetic words are considered (numbers, punctuation, and symbols are ignored).
    • Stopwords (common words like "the", "is", "and") are automatically removed.
  • Keyword Selection:
    • Counts occurrences of words in the text.
    • Selects the top N most frequent words (default: 5 keywords).
  • Output: Extracted keywords are stored in a new column (default: keywords).
  • Visualization: Displays the input text and extracted keywords for review.

‚öôÔ∏è Key Configuration Notes

  • Input Column:
    • Selects the column containing text for keyword extraction.
    • Default: text.
  • Output Column:
    • Defines where extracted keywords are stored.
    • Default: keywords.
  • Number of Keywords:
    • Controls how many keywords are extracted per text.
    • Default: 5, adjustable between 1 and 20.
  • Language Model:
    • Specifies the spaCy language model used for tokenization.
    • Options: en_core_web_md (English) or de_core_news_md (German).

üîß Technical Implementation Details

  • Tokenization: Text is split into individual words using the spaCy NLP pipeline.
  • Stopword Filtering:
    • Common function words (e.g., "the", "and") are ignored.
    • Ensures extracted keywords are meaningful.
  • Frequency Counting:
    • Words are counted, and the most frequent ones are selected.
    • Case-insensitive (e.g., "Apple" and "apple" are treated as the same word).
  • Visualization: Generates an HTML preview showing extracted keywords for a given text.

  • The operation does not understand context, so frequently occurring words may not always be meaningful.
  • Longer texts may produce better keyword results than very short ones.
  • First execution may take longer due to model downloading.
