üõ† How the Sentiment Analysis Operation Works

  • Text Input Processing: The operation processes text data from a specified input column (default: "text").
  • Model Selection:
    • For English (`"en"`): Uses `distilbert-base-uncased-finetuned-sst-2-english`.
    • For German (`"de"`): Uses `oliverguhr/german-sentiment-bert`.
  • Prediction: The model classifies the sentiment of each input text as either POSITIVE or NEGATIVE, along with a confidence score.
  • Truncation and Padding: The model ensures text fits within input limits using truncation and padding.
  • Output: Two new columns are added:
    • `[prefix]_label` - Predicted sentiment label.
    • `[prefix]_score` - Confidence score (0 to 1).
  • Visualization: Displays a sample of the processed text along with its predicted sentiment and confidence score.

üîß Technical Implementation Details

  • Uses Hugging Face Transformers: Implements `transformers.pipeline("sentiment-analysis")`.
  • Batch Processing: Processes input texts efficiently with automatic batching.
  • Score Calculation: The confidence score is rounded to 4 decimal places.
  • Text Sample Handling: Displays up to 400 characters in the visualization, appending `"..."` if truncated.
