üõ† How the Word List Scan Operation Works

  • Text Input Processing: Reads text data from a designated column (default: "text") and prepares it for scanning.
  • Word Lists Configuration: Processes multiple word lists defined in the configuration. Only word lists marked as active are used.
  • Pattern Compilation:
    • For plain words, compiles regex patterns with word boundaries for whole-word matching.
    • For regex entries, compiles the provided pattern directly with case-insensitive matching.
  • Scanning and Counting: Iterates over each active word list, applying all its patterns to the input text and counting the total matches.
  • Visualization: Generates an HTML preview that displays a (possibly truncated) version of the input text alongside the match counts for each word list.

    • Active Flag: Toggle to include or exclude a word list from processing.

üîß Technical Implementation Details

  • Parallelizable Processing: Designed as a parallelizable operation, it efficiently processes each text cell in batch mode with progress updates.
  • Regex Matching: Utilizes Python‚Äôs re module to compile and apply regex patterns with case-insensitive matching.

  • Visualization: Returns an HTML visualization that displays a sample of the input text (truncated to 300 characters if necessary) and the respective match counts.
